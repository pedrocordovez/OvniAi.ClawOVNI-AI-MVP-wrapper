# Plan 1: AWS Infrastructure Setup

<objective>
Provision the full AWS infrastructure: VPC with public/private subnets, security groups, RDS PostgreSQL 16, ElastiCache Redis 7, and an EC2 instance with Docker installed — ready to receive the application deployment.
</objective>

<must_haves>
- VPC with public subnet (EC2) and private subnets (RDS, ElastiCache) in the same AZ
- RDS PostgreSQL 16 instance running and accessible from EC2 only
- ElastiCache Redis 7 node running and accessible from EC2 only
- EC2 instance with Docker, Node.js 20, Git, and PM2 installed
- Security groups correctly restrict access (SSH to Pedro's IP only, 80/443 open, private services locked down)
- Elastic IP assigned to EC2 for stable DNS pointing
</must_haves>

## Tasks

<task id="1.1">
<title>Create VPC with public and private subnets</title>
<read_first>
- .planning/phases/01-aws-infrastructure-deploy/01-CONTEXT.md (decisions D-01 through D-08 for architecture)
</read_first>
<action>
In the AWS Console (or CLI), create the following network infrastructure. Use **us-east-1** (or Pedro's preferred region).

**VPC:**
```
Name: ovni-ai-vpc
CIDR: 10.0.0.0/16
DNS hostnames: Enabled
DNS resolution: Enabled
```

**Subnets (all in the same AZ, e.g. us-east-1a):**
```
Public subnet:
  Name: ovni-ai-public-1a
  CIDR: 10.0.1.0/24
  AZ: us-east-1a
  Auto-assign public IP: Yes

Private subnet 1 (RDS primary):
  Name: ovni-ai-private-1a
  CIDR: 10.0.10.0/24
  AZ: us-east-1a

Private subnet 2 (RDS standby — required for DB subnet group):
  Name: ovni-ai-private-1b
  CIDR: 10.0.11.0/24
  AZ: us-east-1b
```

**Internet Gateway:**
```
Name: ovni-ai-igw
Attach to: ovni-ai-vpc
```

**Route Table (public):**
```
Name: ovni-ai-public-rt
Routes:
  10.0.0.0/16 → local
  0.0.0.0/0 → ovni-ai-igw
Associate with: ovni-ai-public-1a
```

The private subnets use the default route table (no internet access).
</action>
<acceptance_criteria>
- AWS CLI check: `aws ec2 describe-vpcs --filters "Name=tag:Name,Values=ovni-ai-vpc" --query "Vpcs[0].CidrBlock"` returns `"10.0.0.0/16"`
- AWS CLI check: `aws ec2 describe-subnets --filters "Name=tag:Name,Values=ovni-ai-public-1a" --query "Subnets[0].CidrBlock"` returns `"10.0.1.0/24"`
- AWS CLI check: `aws ec2 describe-internet-gateways --filters "Name=tag:Name,Values=ovni-ai-igw"` returns 1 result attached to the VPC
</acceptance_criteria>
</task>

<task id="1.2">
<title>Create security groups</title>
<read_first>
- .planning/phases/01-aws-infrastructure-deploy/01-CONTEXT.md (decision D-03 for SG rules)
</read_first>
<action>
Create three security groups in `ovni-ai-vpc`:

**1. EC2 Security Group:**
```
Name: ovni-ai-ec2-sg
Description: OVNI AI EC2 instance
Inbound rules:
  - SSH (22) from Pedro's IP: <PEDRO_IP>/32
  - HTTP (80) from 0.0.0.0/0
  - HTTPS (443) from 0.0.0.0/0
Outbound: All traffic (default)
```

**2. RDS Security Group:**
```
Name: ovni-ai-rds-sg
Description: OVNI AI RDS PostgreSQL
Inbound rules:
  - PostgreSQL (5432) from ovni-ai-ec2-sg (security group reference)
Outbound: All traffic (default)
```

**3. ElastiCache Security Group:**
```
Name: ovni-ai-redis-sg
Description: OVNI AI ElastiCache Redis
Inbound rules:
  - Custom TCP (6379) from ovni-ai-ec2-sg (security group reference)
Outbound: All traffic (default)
```
</action>
<acceptance_criteria>
- AWS CLI check: `aws ec2 describe-security-groups --filters "Name=group-name,Values=ovni-ai-ec2-sg" --query "SecurityGroups[0].GroupName"` returns `"ovni-ai-ec2-sg"`
- AWS CLI check: `aws ec2 describe-security-groups --filters "Name=group-name,Values=ovni-ai-rds-sg" --query "SecurityGroups[0].GroupName"` returns `"ovni-ai-rds-sg"`
- AWS CLI check: `aws ec2 describe-security-groups --filters "Name=group-name,Values=ovni-ai-redis-sg" --query "SecurityGroups[0].GroupName"` returns `"ovni-ai-redis-sg"`
- RDS SG only allows port 5432 from EC2 SG
- Redis SG only allows port 6379 from EC2 SG
</acceptance_criteria>
</task>

<task id="1.3">
<title>Provision RDS PostgreSQL 16</title>
<read_first>
- docker-compose.yml (current Postgres config: user=ovni, db=ovni_wrapper)
- src/config.ts (DATABASE_URL format)
</read_first>
<action>
Create an RDS DB subnet group first:
```
Name: ovni-ai-db-subnet-group
Subnets: ovni-ai-private-1a, ovni-ai-private-1b
```

Create the RDS instance:
```
Engine: PostgreSQL 16
Instance class: db.t3.micro (upgrade to db.t3.small if needed later)
Identifier: ovni-ai-postgres
Master username: ovni
Master password: <generate a strong password, save securely>
DB name: ovni_wrapper
Storage: 20 GB gp3, autoscaling up to 50 GB
Multi-AZ: No (single instance for cost)
VPC: ovni-ai-vpc
Subnet group: ovni-ai-db-subnet-group
Security group: ovni-ai-rds-sg
Public access: No
Backup retention: 7 days
Deletion protection: Yes
```

After creation, note the endpoint:
```
ovni-ai-postgres.<random>.us-east-1.rds.amazonaws.com
```

The DATABASE_URL will be:
```
postgres://ovni:<password>@ovni-ai-postgres.<random>.us-east-1.rds.amazonaws.com:5432/ovni_wrapper
```
</action>
<acceptance_criteria>
- AWS CLI check: `aws rds describe-db-instances --db-instance-identifier ovni-ai-postgres --query "DBInstances[0].DBInstanceStatus"` returns `"available"`
- RDS engine is PostgreSQL 16: `aws rds describe-db-instances --db-instance-identifier ovni-ai-postgres --query "DBInstances[0].Engine"` returns `"postgres"`
- RDS is NOT publicly accessible: `aws rds describe-db-instances --db-instance-identifier ovni-ai-postgres --query "DBInstances[0].PubliclyAccessible"` returns `false`
- From EC2: `psql postgres://ovni:<password>@<rds-endpoint>:5432/ovni_wrapper -c "SELECT 1"` returns 1
</acceptance_criteria>
</task>

<task id="1.5">
<title>Launch EC2 instance with Docker and Node.js</title>
<read_first>
- .planning/phases/01-aws-infrastructure-deploy/01-CONTEXT.md (decisions D-01, D-02, D-16)
- package.json (engines: node >= 20)
- .env.example (OPENCLAW_VOLUMES_BASE, OPENCLAW_NETWORK)
</read_first>
<action>
**Create an SSH key pair** (if not already existing):
```bash
aws ec2 create-key-pair --key-name ovni-ai-key --query "KeyMaterial" --output text > ovni-ai-key.pem
chmod 400 ovni-ai-key.pem
```

**Launch EC2 instance:**
```
AMI: Ubuntu Server 24.04 LTS (amd64) — latest from AWS Marketplace
Instance type: t3.medium (2 vCPU, 4 GB RAM)
Key pair: ovni-ai-key
VPC: ovni-ai-vpc
Subnet: ovni-ai-public-1a
Security group: ovni-ai-ec2-sg
Storage: 30 GB gp3 root volume
Name tag: ovni-ai-server
```

**Allocate and associate an Elastic IP:**
```bash
aws ec2 allocate-address --domain vpc
# Note the AllocationId
aws ec2 associate-address --instance-id <instance-id> --allocation-id <alloc-id>
# Note the public IP — this is the permanent IP for DNS
```

**SSH in and run setup:**
```bash
ssh -i ovni-ai-key.pem ubuntu@<ELASTIC_IP>

# Update system
sudo apt update && sudo apt upgrade -y

# Install Docker
sudo apt install -y docker.io docker-compose-plugin
sudo systemctl enable docker
sudo systemctl start docker
sudo usermod -aG docker ubuntu

# Install Node.js 20
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# Install PM2 globally
sudo npm install -g pm2

# Install build essentials (for native modules)
sudo apt install -y build-essential

# Install PostgreSQL client (for running migrations and testing RDS)
sudo apt install -y postgresql-client

# Install Redis tools (for testing ElastiCache)
sudo apt install -y redis-tools

# Install Nginx (needed for Plan 3)
sudo apt install -y nginx

# Install Git
sudo apt install -y git

# Create directories for OpenClaw
sudo mkdir -p /var/lib/ovni-ai/instances
sudo chown ubuntu:ubuntu /var/lib/ovni-ai/instances

# Create Docker network for OpenClaw instances
docker network create ovni-ai-instances

# Create app directory
sudo mkdir -p /opt/ovni-ai
sudo chown ubuntu:ubuntu /opt/ovni-ai

# Verify installations
node --version    # v20.x.x
npm --version
docker --version
pm2 --version
nginx -v
git --version
```
</action>
<acceptance_criteria>
- SSH into instance succeeds: `ssh -i ovni-ai-key.pem ubuntu@<ELASTIC_IP>` connects
- Command check: `node --version` outputs `v20.` prefix
- Command check: `docker --version` exits 0
- Command check: `pm2 --version` exits 0
- Command check: `docker network ls | grep ovni-ai-instances` shows the network
- Directory exists: `ls -la /var/lib/ovni-ai/instances` exits 0
- Directory exists: `ls -la /opt/ovni-ai` exits 0
- Elastic IP is associated: `curl -s http://<ELASTIC_IP>` returns Nginx default page (port 80 open)
</acceptance_criteria>
</task>

<task id="1.6">
<title>Test connectivity between EC2, RDS, and ElastiCache</title>
<read_first>
- .env.example (DATABASE_URL and REDIS_URL format)
</read_first>
<action>
SSH into the EC2 instance and verify that all managed services are reachable:

```bash
ssh -i ovni-ai-key.pem ubuntu@<ELASTIC_IP>

# Test RDS PostgreSQL connectivity
psql "postgres://ovni:<RDS_PASSWORD>@<RDS_ENDPOINT>:5432/ovni_wrapper" -c "SELECT version();"
# Expected: PostgreSQL 16.x

# Test ElastiCache Redis connectivity
redis-cli -h <ELASTICACHE_ENDPOINT> -p 6379 ping
# Expected: PONG

# Test Docker works
docker run --rm hello-world
# Expected: "Hello from Docker!"

# Test Docker network
docker network inspect ovni-ai-instances
# Expected: JSON output showing the network
```

Create the production `.env` file at `/opt/ovni-ai/.env`:
```bash
cat > /opt/ovni-ai/.env << 'ENVEOF'
# ─── Server ─────────────────────────────────────────────
PORT=3000
NODE_ENV=production

# ─── Database ───────────────────────────────────────────
DATABASE_URL=postgres://ovni:<RDS_PASSWORD>@<RDS_ENDPOINT>:5432/ovni_wrapper
REDIS_URL=redis://<ELASTICACHE_ENDPOINT>:6379

# ─── Anthropic ──────────────────────────────────────────
ANTHROPIC_API_KEY=<your-anthropic-key>
ANTHROPIC_MARGIN=1.25

# ─── Email (Resend) ────────────────────────────────────
RESEND_API_KEY=<your-resend-key>
SMTP_FROM=noreply@ovni.ai
OPS_ALERT_EMAIL=ops@ovni.ai

# ─── Stripe ────────────────────────────────────────────
STRIPE_SECRET_KEY=<your-stripe-key>
STRIPE_WEBHOOK_SECRET=<your-stripe-webhook-secret>

# ─── Twilio / WhatsApp ─────────────────────────────────
# TWILIO_ACCOUNT_SID=AC...
# TWILIO_AUTH_TOKEN=...
# TWILIO_WHATSAPP_FROM=whatsapp:+1...

# ─── OpenClaw Orchestration ────────────────────────────
OPENCLAW_IMAGE=openclaw/openclaw:latest
OPENCLAW_VOLUMES_BASE=/var/lib/ovni-ai/instances
OPENCLAW_NETWORK=ovni-ai-instances

# ─── Metering Proxy ────────────────────────────────────
METERING_PROXY_PORT=3001
METERING_PROXY_URL=http://localhost:3001

# ─── API Key Vault encryption ──────────────────────────
VAULT_ENCRYPTION_KEY=<generate-with: node -e "console.log(require('crypto').randomBytes(32).toString('hex'))">
ENVEOF
```

Replace all `<placeholder>` values with actual credentials. Set file permissions:
```bash
chmod 600 /opt/ovni-ai/.env
```

Note: `METERING_PROXY_URL` is `http://localhost:3001` (not `host.docker.internal`) because the app runs directly on the host, not in a container. OpenClaw containers will use the EC2 private IP to reach the metering proxy.
</action>
<acceptance_criteria>
- RDS returns PostgreSQL 16 version string
- Redis returns PONG
- Docker hello-world succeeds
- File exists: `/opt/ovni-ai/.env` with permissions 600
- `.env` file contains `NODE_ENV=production`
- `.env` file contains `DATABASE_URL=postgres://ovni:` prefix
- `.env` file contains `REDIS_URL=redis://` prefix
- `.env` file contains `METERING_PROXY_URL=http://localhost:3001`
</acceptance_criteria>
</task>
