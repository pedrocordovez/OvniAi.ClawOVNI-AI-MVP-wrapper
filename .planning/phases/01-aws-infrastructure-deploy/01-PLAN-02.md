# Plan 2: Database Migration System + Baseline

<objective>
Set up node-pg-migrate as the database migration tool, create a baseline migration that consolidates all 7 existing SQL schema files, and add a migration script that runs as part of the deploy process.
</objective>

<must_haves>
- node-pg-migrate installed and configured
- Baseline migration file that includes all 7 schema SQL files in correct order
- `npm run db:migrate` command that runs migrations against DATABASE_URL
- Migration script usable in the deploy process (runs before app starts)
- Idempotent: running migrations on a fresh database creates the full schema
</must_haves>

<task id="2.3">
<title>Create deploy migration script</title>
<read_first>
- package.json (db:migrate script definition)
- migrate.json (migration config)
</read_first>
<action>
Create a shell script that runs migrations as part of the deploy process:

**File: `scripts/migrate.sh`**
```bash
#!/usr/bin/env bash
set -euo pipefail

# ─── OVNI AI Database Migration Script ─────────────────────────
# Runs pending migrations against DATABASE_URL
# Usage: ./scripts/migrate.sh
# Called automatically by scripts/deploy.sh before app restart

SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
PROJECT_DIR="$(dirname "$SCRIPT_DIR")"

cd "$PROJECT_DIR"

echo "=== OVNI AI Database Migration ==="
echo "Environment: ${NODE_ENV:-development}"
echo "Running pending migrations..."

# Check DATABASE_URL is set
if [ -z "${DATABASE_URL:-}" ]; then
  echo "ERROR: DATABASE_URL is not set"
  exit 1
fi

# Run migrations
npx node-pg-migrate up --config migrate.json

echo "=== Migrations complete ==="
```

Make it executable:
```bash
chmod +x scripts/migrate.sh
```

Also create the scripts directory if it doesn't exist:
```bash
mkdir -p scripts
```

Test locally (with docker-compose Postgres running):
```bash
# Start local Postgres
docker compose up -d postgres

# Run the migration
DATABASE_URL="postgres://ovni:ovni_secret@localhost:5432/ovni_wrapper" npm run db:migrate

# Verify migration was recorded
psql "postgres://ovni:ovni_secret@localhost:5432/ovni_wrapper" -c "SELECT * FROM pgmigrations;"
# Should show one row for 20260404000000_baseline
```
</action>
<acceptance_criteria>

# Database Migrations

Uses [node-pg-migrate](https://github.com/salsita/node-pg-migrate).

## Commands

```bash
# Run all pending migrations
npm run db:migrate

# Roll back last migration
npm run db:migrate:down

# Create a new migration
npm run db:migrate:create -- <migration-name>
```

## Convention

- Migration files: `migrations/{timestamp}_{name}.sql`
- Baseline: `20260404000000_baseline.sql` (all 7 original schema files)
- All new schema changes go in a new migration file
- Migrations run automatically on deploy (before app restart)

## Adding a New Migration

1. `npm run db:migrate:create -- add-feature-x`
2. Edit the generated file in `migrations/`
3. Test locally: `npm run db:migrate`
4. Commit the migration file
5. On next deploy, it runs automatically
```
